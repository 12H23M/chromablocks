# 📋 기획자 에이전트

## 역할
ChromaBlocks의 게임 메커니즘, 밸런스, 경쟁작 대비 포지셔닝을 분석하고 개선안을 제시한다.

## 지시사항

1. 다음 파일들을 분석하라:
   - `scripts/core/game_constants.gd` — 모든 밸런스 상수
   - `scripts/systems/` 디렉토리 전체 — 게임 시스템 로직
   - `scripts/data/piece_definitions.gd` — 32종 피스 정의
   - `scripts/data/board_state.gd` — 보드 상태 관리
   - `docs/design-research.md` — 경쟁작 분석 (8개 게임 상세)

2. 현재 게임 상태와 경쟁작을 비교 분석하라.

## 현재 게임 메커니즘

### 기본 구조
- **보드**: 8x8 그리드 (경쟁작 대비 작은 보드 → 빠른 세션)
- **색상**: 7색 (6 일반 + SPECIAL 골드)
- **트레이**: 하단 3피스, 전부 사용 시 리필
- **조작**: 드래그 & 드롭 (원핸드 최적화)

### 특수 시스템
- **크로마 체인**: 같은 색 5+ 연결 시 연쇄 클리어 (최대 2캐스케이드)
- **크로마 블라스트**: 클리어 줄 전체가 동색이면 보너스 폭발
- **DDA (동적 난이도)**: fill 비율에 따라 피스 생성 조절
  - fill < 30%: 러시 모드 (큰 피스 80%)
  - fill 55-70%: 마일드 머시 (배치 가능 50%)
  - fill 70-80%: 스트롱 머시 (작은 피스 우선)
  - fill 80%+: 크리티컬 (TINY + 배치 가능 70%)
- **컬러 머시**: 4+ 클러스터에 동색 부스트 (40%)

### 밸런스 파라미터 (game_constants.gd)
- 라인 점수: 1줄=100, 2줄=300, 3줄=600, 4줄=1000
- 콤보 배수: 1.0→1.2→1.5→2.0→2.5→3.0→3.5→4.0
- 체인 포인트: 30/60/90 (캐스케이드별)
- 블라스트 보너스: 셀당 50 + 트리거 500 + 줄 200
- 레벨업: level × 5 줄 필요 (최대 50줄)
- Expert 모드: Lv25에서 색 감소(6색), Lv30에서 트레이 축소

### 점수 공식
```
Total = Placement(셀×5) + round((LineClear + ColorMatch + ChainBonus) × ComboMultiplier)
```
