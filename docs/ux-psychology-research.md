## 1. 주스(Juice) 이론

### 핵심 개념
"Juice"란 플레이어의 입력에 대한 **과장되고 즉각적인 피드백**을 말한다. Martin Jonasson & Petri Purho의 GDC 발표 "Juice It or Lose It"(2012)에서 체계화되었다. 같은 브레이크아웃 게임이라도 주스의 유무에 따라 플레이 시간이 **3~5배** 차이난다.

### "한판만 더"를 유발하는 핵심 주스 요소

| 요소 | 효과 | ChromaBlocks 적용 |
|------|------|-------------------|
| **Tweening (이징)** | 모든 움직임에 ease-out 적용. 딱딱한 선형 이동은 "싸구려" 느낌 | 블록 배치, 슬라이드, 낙하 모두 `easeOutBack` 또는 `easeOutElastic` |
| **Screenshake** | 임팩트 순간의 화면 떨림. Jan Willem Nijman의 "Art of Screenshake"(2013) 핵심 | 줄 클리어 시 미세한 shake (2-4px, 100-150ms). 멀티라인은 강도 증가 |
| **Squash & Stretch** | 블록이 착지할 때 살짝 찌그러졌다 복원. 물리적 무게감 | 블록 착지 시 scaleY 0.85→1.0 (80ms), scaleX 1.1→1.0 |
| **파티클** | 클리어, 콤보 시 튀어나오는 조각들. 시각적 "폭발"감 | 클리어된 블록 색상과 동일한 작은 사각형 파티클 8-16개 |
| **Trail/잔상** | 빠르게 움직이는 오브젝트 뒤의 잔상 | 블록 드래그 시 반투명 트레일 |
| **Flash** | 임팩트 순간 흰색 플래시 (1-2프레임) | 줄 클리어 순간 해당 줄 밝기 200% → 페이드아웃 |
| **카메라 줌** | 중요 이벤트 시 살짝 줌인→복귀 | 콤보 시 scale 1.02→1.0 (200ms) |

### 주스의 황금 법칙
1. **입력 즉시 반응**: 지연 0ms. 터치 시 블록이 즉시 반응 (하이라이트, 살짝 커짐)
2. **과장하되 짧게**: 이펙트는 화려하지만 150-300ms 이내 종료
3. **누적 강화**: 콤보가 쌓일수록 이펙트 강도 증가 → 도파민 경사 형성
4. **소리와 동기화**: 시각 이펙트만으로는 50%. 사운드와 결합 시 만족감 2배

## 2. 색상 심리학

### 블록 퍼즐에서 색상이 감정에 미치는 영향

#### 색상별 심리적 효과 (게임 맥락)

| 색상 | 감정/연상 | 게임 내 역할 | 주의사항 |
|------|-----------|-------------|---------|
| **빨강** (#E74C3C) | 긴급, 흥분, 위험 | 에너지/파워 블록, 점수 부스터 | 과다 사용 시 스트레스 유발 |
| **파랑** (#3498DB) | 안정, 신뢰, 차분 | 기본/안전 블록 | 주 색상으로 적합, 눈 피로 적음 |
| **초록** (#2ECC71) | 성장, 긍정, 자연 | 보상/진행 표시 | 빨강과 함께 쓸 때 색약 주의 |
| **노랑** (#F1C40F) | 행복, 낙관, 주목 | 특수 블록, 하이라이트 | 밝은 배경에서 가시성 낮음 |
| **보라** (#9B59B6) | 고급, 미스터리, 창의 | 레어/특수 아이템 | "프리미엄" 느낌 부여에 효과적 |
| **주황** (#E67E22) | 열정, 에너지 | 콤보/멀티플라이어 | 빨강과 혼동 가능 → 채도 차이 필요 |
| **청록** (#1ABC9C) | 세련, 현대적 | UI 악센트 색상 | 다크 테마에서 특히 세련된 느낌 |

#### 블록 퍼즐 색상 배치 원칙

1. **인접 색상 회피**: 색상환에서 최소 60° 이상 떨어진 색상 사용
2. **채도 통일**: 모든 블록의 채도(Saturation)를 동일 레벨로 맞추면 "세트감" 형성
3. **명도 차별화**: 같은 채도라도 명도(Brightness)로 구분 용이성 확보
4. **따뜻한 색 < 차가운 색**: 비율 4:6 권장. 차가운 색이 더 편안한 장시간 플레이 유도

### 테트리스/블록 퍼즐 레퍼런스 분석

- **테트리스 이펙트**: 네온 톤 + 다크 배경 → 몰입감 극대화
- **1010!**: 낮은 채도 파스텔 → 캐주얼/릴렉스 느낌
- **우드블록 퍼즐**: 따뜻한 우드 톤 → 친근/아날로그 감성
- **블록 블라스트**: 비비드 컬러 + 강한 대비 → 아케이드적 흥분

## 3. 시각적 보상 체계

### 보상 이벤트별 이펙트 강도 가이드

#### 계층적 보상 피라미드 (Skinner의 변동 비율 강화 적용)

```
         ★ 레벨업/퍼펙트 클리어 ★         ← Tier 4: 드문 대형 보상
        ━━━━━━━━━━━━━━━━━━━━━━
       🔥 콤보 3x 이상 / 멀티라인 🔥      ← Tier 3: 기대 이상의 보상
      ━━━━━━━━━━━━━━━━━━━━━━━━━━
     ✦ 2줄 동시 클리어 / 콤보 시작 ✦       ← Tier 2: 스킬풀한 플레이 보상
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ● 1줄 클리어 / 블록 배치 ●               ← Tier 1: 기본 만족
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 각 Tier별 이펙트 상세

**Tier 1: 기본 만족 (1줄 클리어)**
- 클리어된 줄이 밝아지며 → 양쪽으로 dissolve (200ms)
- 위 블록이 부드럽게 낙하 (easeOutBounce, 300ms)
- 점수 +N 텍스트 팝업 (scale 0→1.2→1.0, 500ms 후 페이드)
- 미세한 haptic (모바일)
- 화면 흔들림: 없음 또는 1px

**Tier 2: 스킬풀 (2줄 / 콤보 시작)**
- Tier 1 + 클리어 라인에서 빛 웨이브 방사 (좌→우 sweep)
- "Nice!" 텍스트 (노랑 그라디언트, bounce 애니메이션)
- 파티클 16개 (블록 색상별)
- 화면 흔들림: 2px, 100ms
- 콤보 카운터 등장 + 펄스 애니메이션

**Tier 3: 기대 이상 (3줄+ / 콤보 3x+)**
- Tier 2 + 화면 전체 밝기 플래시 (50ms)
- 클리어 영역에서 방사형 파티클 폭발 (32개+)
- "Amazing!" / "Combo x3!" 텍스트 (큰 폰트, 그라디언트 + glow)
- 배경색 순간 변화 (subtle pulse)
- 화면 흔들림: 3-4px, 150ms
- 점수 텍스트 크기 + 색상 강화

**Tier 4: 극히 드문 (레벨업 / 퍼펙트)**
- 전체 화면 서서히 화이트아웃 → 복귀 (500ms)
- 중앙에 대형 축하 텍스트 + 아이콘
- 컨페티/불꽃놀이 파티클 (3-5초 지속)
- 배경 그라디언트 전환 (새 레벨 테마)
- 모든 남은 블록에 순차 glow 웨이브
- 강한 haptic 패턴

### 변동 보상의 핵심 (Variable Ratio Schedule)

Skinner Box 이론의 게임 적용:
- **고정 보상**은 예측 가능 → 빠르게 무감각해짐
- **변동 보상**이 도파민 시스템을 활성화
- ChromaBlocks 적용:
  - 같은 3줄 클리어라도 **가끔** 보너스 이펙트 추가 (10-15% 확률)
  - "럭키 클리어" 이펙트: 무지개 파티클 + 보너스 점수
  - 연속 플레이 시 점진적으로 이펙트 강화 → 이탈 방지

## 4. 인지 부하 최적화

### 최적 블록/색상 수

#### Miller의 법칙과 Hick의 법칙 적용

- **Miller의 법칙**: 인간의 작업 기억 용량 = 7±2 항목
- **Hick의 법칙**: 선택지가 많을수록 결정 시간이 로그적으로 증가
- **블록 퍼즐 최적값**: 색상 **5-6개**가 최적

| 색상 수 | 인지 부하 | 게임성 | 평가 |
|---------|----------|--------|------|
| 3-4개 | 매우 낮음 | 너무 쉬움, 빠르게 지루 | ❌ |
| **5개** | **적절** | **캐주얼 유저 최적** | ✅ 권장 (시작) |
| **6개** | **적절~중간** | **숙련 유저 도전감** | ✅ 권장 (진행 후) |
| 7개 | 높음 | 색 구분 어려움 시작 | ⚠️ 상급 레벨만 |
| 8개+ | 과부하 | 좌절감, 이탈 | ❌ |

#### 점진적 색상 도입 전략
```
레벨  1-5:   4색 → 기본 학습
레벨  6-15:  5색 → 최적 플레이
레벨 16-30:  6색 → 도전 강화
레벨 31+:    6색 유지 + 특수 블록으로 복잡도 추가
```

### 색 구분 용이성

#### 최적 색상 간 거리 (CIELAB 색공간 기준)
- 최소 ΔE ≥ 25 (일반 시력)
- 색약 접근성: ΔE ≥ 40 + 명도 차이 + 패턴/형태 보조

#### 구분을 돕는 보조 시각 요소
1. **형태 차별화**: 각 색상 블록에 미묘하게 다른 내부 패턴 (대각선, 점, 격자 등)
2. **명도 단계화**: 밝은 것부터 어두운 것까지 고르게 분포
3. **색약 모드**: 블록에 기호(●, ▲, ■, ◆, ★) 오버레이 옵션
4. **테두리 색상**: 본체보다 약간 어두운 테두리로 분리감

## 5. 플로우 상태 유도

### Csikszentmihalyi의 Flow 조건과 시각 디자인

Flow의 8가지 조건 중 시각 디자인과 직결되는 4가지:

#### 1. 명확한 목표의 시각화
- 목표 진행률을 **항상 시야에** 두되, **주의를 빼앗지 않게**
- 프로그레스 바: 화면 최상단, 얇고(3-4px), 부드러운 색상
- 레벨 번호: 작지만 항상 보이는 곳

#### 2. 즉각적 피드백
- 블록 터치 → 0ms 반응 (하이라이트)
- 유효 배치 위치 → 고스트 블록 표시
- 무효 배치 → 미세한 붉은 플래시 + 가벼운 바운스백
- **피드백 지연 100ms 이상 시 플로우 파괴**

#### 3. 도전과 능력의 균형 (시각적 표현)
- 보드 30% 이하 차있을 때: 배경 평화로움, 음악 차분
- 보드 50-70% 차있을 때: 배경 미묘한 색조 변화 (약간 따뜻해짐)
- 보드 80%+ 차있을 때: 경고 표시 (가장자리 미세한 붉은 바이닝 + 펄스)
- **절대로 갑작스러운 변화 금지** → 점진적 전환이 핵심

#### 4. 집중 방해 요소 제거
- **게임 중 팝업/배너 절대 금지**
- HUD 요소 최소화 (점수, 레벨만)
- 광고는 게임 오버/레벨 클리어 후에만
- 알림은 자동으로 비활성화 (fullscreen 모드 활용)

### 플로우 파괴자 TOP 5 (피해야 할 것)
1. ❌ 튜토리얼 팝업이 게임 중간에 뜸
2. ❌ 점수 계산이 길어서 다음 움직임 대기
3. ❌ 너무 화려한 이펙트가 다음 수 판단을 가림
4. ❌ UI 요소가 게임 영역을 가림
5. ❌ 로딩/전환 시간 > 0.5초

## 6. 첫인상 (FTUE - First Time User Experience)

### 앱 시작 5초 안에 잡아야 하는 것

#### 5초 타임라인

```
0.0s ─ 앱 실행
0.0-0.5s ─ 로고/스플래시 (고급스러운 로고 애니메이션)
0.5-1.5s ─ 메인 화면 전환 (블록들이 떨어지며 조립되는 인트로 모션)
1.5-3.0s ─ "Play" 버튼 + 블록 배경 미리보기
3.0-5.0s ─ 첫 터치 → 즉각 게임 시작 (튜토리얼 없이 바로)
```

#### "이 게임 재밌겠다" 만드는 비주얼 핵심 5가지

1. **모션이 있는 스플래시**: 로고가 블록으로 조립되는 애니메이션 → "이 게임은 폴리시 있다"
2. **즉시 만질 수 있는 것**: 메인 화면에서 이미 블록을 터치하면 반응 (파티클, 색상 변화)
3. **색상 팔레트 첫인상**: 블록 색상이 **조화롭고 세련되면** = 프리미엄 게임 인식
4. **빈 보드가 아닌 보드**: 첫 게임 시작 시 몇 개의 블록이 이미 놓여있는 상태 → "여기서부터 시작" 안내 역할
5. **소리**: 앱 시작 시 부드러운 앰비언트 사운드 → 몰입 시작 신호

#### FTUE 안티패턴 (절대 하지 말 것)
- ❌ 로그인/회원가입 강제
- ❌ 긴 튜토리얼 (3단계 이내로)
- ❌ 권한 요청 팝업 (나중에)
- ❌ 업데이트/공지 팝업
- ❌ 빈 화면에서 "시작" 버튼만 덩그러니

## 7. 다크 테마 vs 라이트 테마

### 연구 결과 요약

| 기준 | 다크 테마 | 라이트 테마 |
|------|----------|------------|
| **장시간 플레이** | ✅ 눈 피로 감소 → 더 오래 플레이 | ❌ 30분+ 시 피로 증가 |
| **색상 대비** | ✅ 블록 색상이 돋보임 | ⚠️ 밝은 색 블록이 묻힘 |
| **프리미엄 느낌** | ✅ 고급스러움 | ⚠️ 캐주얼/가벼운 느낌 |
| **OLED 배터리** | ✅ 전력 절약 | ❌ 더 많은 전력 |
| **주간 가시성** | ⚠️ 밝은 환경에서 약간 불리 | ✅ 밝은 곳에서 잘 보임 |
| **감성** | 몰입적, 시네마틱 | 경쾌, 친근 |

### 결론: **다크 테마 기본, 라이트 옵션 제공**

ChromaBlocks 권장:
- **기본값: 다크 테마** (배경 #1A1A2E 또는 #0F0F23)
- 블록 색상이 어두운 배경에서 **보석처럼** 빛나는 효과
- 라이트 테마는 설정에서 전환 가능 (파스텔 톤 블록으로 자동 조정)
- **다크 테마 배경은 순수 검정(#000) 피하기** → 너무 강한 대비로 눈 피로
- 권장 배경: 매우 어두운 네이비/차콜 (#16213E, #1A1A2E)

### 다크 테마에서 블록을 빛나게 만드는 법
1. **Inner glow**: 블록 내부에서 바깥으로 미세한 발광
2. **Subtle gradient**: 단색이 아닌 미세 그라디언트 (위→아래 5-10% 밝기 차)
3. **Edge highlight**: 상단/좌측 모서리에 1px 하이라이트 라인
4. **Shadow**: 블록 아래/오른쪽 2-3px 그림자로 입체감

#### 색약 접근성

- 코랄 레드 ↔ 민트 그린: 명도 차이로 구분 가능 (빨강이 더 밝음)
- 추가 안전장치: 각 블록에 **미세 패턴** 오버레이

### 2. 클리어 이펙트 강도 가이드

#### 핵심 원칙: **만족스럽되 다음 수를 가리지 않는 선**

```
이펙트 강도 스펙트럼:

부족 ←──────────────────────────→ 과잉
      ↑ 밋밋, 성취감 없음   시각 방해, 짜증 ↑
                    ↑
              ★ SWEET SPOT ★
         "우와!" + 즉시 다음 수 가능
```

#### 구체적 수치 가이드

| 이벤트 | 지속 시간 | 파티클 수 | 화면 흔들림 | 텍스트 | 딜레이 |
|--------|----------|----------|------------|--------|--------|
| 블록 배치 | 80ms | 0 | 0px | 없음 | 0ms |
| 1줄 클리어 | 200ms | 4-8 | 0-1px | "+100" (작게) | 0ms |
| 2줄 클리어 | 300ms | 12-16 | 2px | "+250 Nice!" | 0ms |
| 3줄 클리어 | 400ms | 24-32 | 3px | "+500 Amazing!" | 50ms |
| 4줄+ 클리어 | 500ms | 48+ | 4px | "+1000 INCREDIBLE!" | 100ms |
| 콤보 x2 | 250ms | 8 | 1px | "Combo x2" | 0ms |
| 콤보 x5+ | 400ms | 32+ | 3px | "COMBO x5!" (큰 폰트) | 50ms |
| 레벨업 | 1500ms | 100+ (컨페티) | 5px | 레벨 번호 대형 표시 | 500ms |
| 게임오버 | 2000ms | 0 | 0px | 최종 점수 | 1000ms |

#### 이펙트 레이어링 규칙
1. **기본 이펙트는 게임플레이를 블록하지 않음** (1-2줄 클리어 중에도 조작 가능)
2. **3줄+ 클리어만 짧은 일시정지** (50-100ms → 임팩트 강조)
3. **파티클은 게임 영역 위에 오버레이** (블록 아래로 가지 않음)
4. **모든 이펙트에 `will-change: transform` + GPU 가속** (프레임 드랍 방지)

### 3. HUD 정보량 최적화

#### 보여줘야 할 것 (항상 표시)

```
┌─────────────────────────────┐
│  Lv.12     ████████░░ 80%   │  ← 레벨 + 진행률 바 (최상단, 미니멀)
│                             │
│                             │
│       [게임 보드]            │
│                             │
│                             │
│  Score: 12,450              │  ← 점수 (좌하단, 중간 크기)
│  [다음 블록 미리보기 x3]     │  ← 하단 또는 우측
└─────────────────────────────┘
```

**항상 표시 (3가지만)**
1. **점수**: 현재 세션의 성취감
2. **레벨/진행률**: 목표 의식
3. **다음 블록**: 전략 수립에 필수

#### 숨겨야 할 것

| 정보 | 이유 | 대안 |
|------|------|------|
| 콤보 카운터 | 발생할 때만 필요 | 콤보 시에만 팝업 → 사라짐 |
| 최고 점수 | 게임 중 비교는 스트레스 | 게임 오버 화면에서만 |
| 시간 | 시간 인식 = 이탈 촉발 | **절대 표시 금지** |
| 설정 버튼 | 게임 중 거의 안 씀 | 일시정지 메뉴 안에 |
| 통계 | 정보 과부하 | 메인 메뉴/결과 화면 |
| 코인/재화 | 게임 플레이와 무관 | 메인 메뉴에서만 |

#### HUD 디자인 원칙
1. **불투명도 80%**: HUD 요소는 약간 반투명 → 보이되 주장하지 않음
2. **Sans-serif 폰트**: 가독성 최우선 (Inter, SF Pro, Pretendard 등)
3. **점수 변화 시에만 강조**: 평상시 조용히, 점수 올라갈 때만 잠깐 반짝
4. **세로 여백 충분히**: 게임 보드와 HUD 사이 최소 16px 간격

### 4. "프리미엄 게임" 느낌을 주는 미세한 디테일 10가지

#### 1. 🫧 블록에 미세한 내부 그라디언트
단색 블록 → 상단이 5-10% 밝은 그라디언트. 이것만으로 "플라스틱 장난감" → "보석" 변환.
```css
background: linear-gradient(180deg, 
  rgba(255,255,255,0.15) 0%, 
  rgba(0,0,0,0.1) 100%);
```

#### 2. ✨ 보드 위 미세한 앰비언트 파티클
아무 일 없을 때도 1-2초에 한 번씩 미세한 빛나는 점이 떠다님. "살아있는" 느낌. 속도 극히 느리게 (2-3초에 화면 1/4 이동), 불투명도 20-30%.

#### 3. 🎵 블록 배치 시 음계 사운드
블록을 놓을 때마다 음계가 올라감 (도→레→미...). 연속 배치가 멜로디가 됨. 테트리스 이펙트가 이걸로 극찬받음.

#### 4. 🔲 블록 모서리 라운딩 (3-4px)
완전한 사각형 → 캐주얼/레트로. 약간 둥근 모서리 → 현대적/프리미엄. 과하게 둥글면(8px+) 장난감 같아짐.

#### 5. 💫 터치 시 블록 미세 확대 (scale 1.05)
손가락이 블록에 닿는 순간 5% 커짐 + 그림자 추가. 100ms ease-out. "이 블록을 잡았다"는 촉각적 피드백의 시각화.

#### 6. 🌊 줄 클리어 시 위 블록 "물결" 낙하
위 블록들이 동시에 뚝 떨어지지 않고, 좌→우로 0.03초 간격으로 순차 낙하. 도미노 같은 만족감.

#### 7. 🎨 배경 그라디언트 미세 변화
레벨이 올라갈 때마다 배경색이 **극히 미묘하게** 변화 (10초에 걸쳐 전환). 플레이어가 의식하지 못하지만 신선함을 느낌.

#### 8. 📐 그리드 라인의 존재감
그리드는 보여야 하지만 주장하면 안 됨. 불투명도 5-8%의 가는 선. 블록이 놓이면 해당 칸의 그리드만 살짝 밝아짐.

#### 9. 🏆 점수 카운터의 "회전 숫자" 효과
점수가 올라갈 때 숫자가 슬롯머신처럼 위로 회전하며 바뀜. CSS `transform: translateY` 애니메이션. 단순 숫자 교체보다 10배 만족스러움.

#### 10. 💎 게임 오버 화면의 "이번 판 하이라이트"
게임 오버 시 "이번 판에서 당신은..." → 최대 콤보, 클리어한 총 줄 수, 가장 많이 쓴 블록 색상 등을 시각적으로 보여줌. 플레이어가 "다음엔 더 잘하자" → 재플레이.
