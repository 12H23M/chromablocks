## 1. 핵심 진단 요약

**한 문장 진단:** ChromaBlocks는 "잘 만든 퍼즐 엔진"이지만 "중독시키는 감정 설계"가 없다.

### 근본 원인 3가지

| 순위 | 원인 | 심리학 이론 | 심각도 |
|------|------|-----------|--------|
| 🥇 | **보상 루프의 평탄함** — 모든 보상이 예측 가능하고 동질적 | Skinner의 Variable Ratio Schedule 부재 | ★★★★★ |
| 🥈 | **목표 부재 (What's Next?)** — 클리어 후 "그래서 뭘 하라고?" | 플로우의 명확한 목표 조건 미충족 | ★★★★☆ |
| 🥉 | **게임오버 시 Near-Miss 부재** — "아 아까워!" 대신 "뭐, 끝났네" | Kahneman의 손실 회피 + Near-miss 효과 미작동 | ★★★★☆ |

## 2. 플로우 상태 진단

Csikszentmihalyi의 Flow 8가지 조건을 ChromaBlocks에 대입한다.

### 2.1 조건별 진단표

| # | Flow 조건 | ChromaBlocks 현황 | 충족? | 분석 |
|---|----------|------------------|------|------|
| 1 | **명확한 목표** | 점수 올리기, 레벨업... 그런데 **왜?** | ⚠️ 부분 | 점수는 숫자일 뿐. "이번 판에서 뭘 달성해야 하는가"가 불명확 |
| 2 | **즉각적 피드백** | 블록 배치, 라인 클리어 이펙트 잘 됨 | ✅ 충족 | hit-stop, screenshake, 파티클, 사운드 — 기술적으로 우수 |
| 3 | **도전/능력 균형** | DDA 시스템 있음. 하지만... | ⚠️ 부분 | 레벨 초반 너무 쉽고, 갑자기 어려워지는 커브. "적절한 긴장감"이 지속되는 구간이 짧음 |
| 4 | **통제감** | Hold 기능, 3피스 미리보기 있음 | ✅ 충족 | 전략적 선택지 적절. 단, 색상 랜덤이 가끔 답없음 |
| 5 | **집중 (몰입)** | UI 미니멀, 게임 중 팝업 없음 | ✅ 충족 | 기본은 좋지만, 몰입할 "이유"가 약함 |
| 6 | **시간 왜곡** | ❌ 발생하지 않음 | ❌ 미충족 | **핵심 문제.** 시간 왜곡은 플로우의 결과이자 지표. 안 일어나면 플로우 상태가 아닌 것 |
| 7 | **자아 의식 상실** | 전략 게임이라 어느 정도 인지적 참여 있음 | ⚠️ 부분 | 분석적 사고가 필요한 게임이라 "무의식적 몰입"보다는 "의식적 참여" |
| 8 | **내적 보상** | 점수 자체에 의미 부여 부족 | ❌ 미충족 | "이 점수가 나한테 뭘 의미하지?" — 리더보드도 없고, 친구 비교도 없음 |

### 2.2 Flow Channel 분석

```
도전 레벨
    ↑
    │          ╱ ANXIETY (불안/좌절)
    │        ╱
    │      ╱     ← ChromaBlocks 후반 (보드 75%+)
    │    ╱  ★ FLOW ZONE ★
    │  ╱         ← ChromaBlocks가 여기 머무는 시간이 너무 짧음
    │╱
    │    BOREDOM (지루)
    │         ← ChromaBlocks 초반 (레벨 1-5, 보드 30% 프리필)
    └─────────────────→ 능력 레벨
```

**문제:** 플로우 존에 머무는 시간이 짧다. 게임이 시작되면 프리필 보드(30%)가 있지만 쉽게 클리어되고, 한동안 "쌓기만 하는" 지루한 구간이 온다. 그러다 갑자기 보드가 빽빽해지면 불안으로 전환. "적절한 긴장감"의 구간이 너무 짧다.

### 2.3 코드 근거

`difficulty_system.gd`의 레벨 계산:
```gdscript
# 레벨업 = 누적 라인 클리어 기반
static func calculate_level(total_lines_cleared: int) -> int
```

`game_constants.gd`의 레벨업 요구량:
```gdscript
static func lines_for_next_level(level: int) -> int:
    return mini(level * 5, 50)
# 레벨 1: 5줄, 레벨 2: 10줄, ..., 레벨 10+: 50줄
```

**문제점:** 레벨업 곡선이 단조롭다. 레벨 1은 5줄이면 되지만 곧바로 급격히 증가한다. 레벨업 자체도 게임플레이에 미치는 영향이 미미하다 (레벨 25 전까지 색상 수 변화 없음, 레벨 30 전까지 트레이 변화 없음).

## 3. 보상 루프 분석

### 3.1 Core Loop 해부

```
┌────────────────────────────────────────────────┐
│                                                │
│  [피스 선택] → [배치] → [클리어?] → [점수+] → │
│       ↑                                ↓       │
│       └── [트레이 리필] ← [3피스 소진] ←┘       │
│                                                │
│  그런데... 점수가 올라가면 뭐가 달라지는가?     │
│  ❌ 아무것도.                                   │
│                                                │
└────────────────────────────────────────────────┘
```

### 3.2 현재 보상 시스템의 문제

#### A. 점수 — "숫자만 올라감"

`scoring_system.gd` 분석:

```gdscript
# 배치: 셀당 5점 (고정)
var placement := cell_count * GameConstants.PLACEMENT_POINTS_PER_CELL  # 5

# 라인 클리어: {1: 100, 2: 300, 3: 600, 4: 1000} (고정)
var line_clear := GameConstants.line_clear_score(clear_result.get("lines_cleared", 0))

# 콤보 배율: [1.0, 1.2, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0] (고정 테이블)
```

**심리학적 문제:**
- 모든 보상이 **완전히 예측 가능** (Fixed Ratio Schedule)
- Skinner Box 실험에서 Fixed Ratio는 **가장 빠르게 포화(satiation)가 오는** 스케줄
- 1줄 클리어 = 항상 100점. 변동 없음. 10번째나 100번째나 같은 느낌

#### B. 레벨 — "의미 없는 숫자"

레벨이 올라가도:
- 레벨 1-24: 아무 변화 없음 (색상 6개 고정, 트레이 3개 고정)
- 레벨 25: 색상이 줄어듦 (오히려 쉬워짐?)
- 레벨 30: 트레이 2개로 감소

**문제:** 레벨업은 "축하 팝업"만 있을 뿐, 게임 경험에 의미 있는 변화를 주지 않는다. 플레이어가 레벨업을 추구할 동기가 없다.

#### C. 업적 — "체크리스트 식상함"

`achievement_system.gd` 분석:

```gdscript
# 21개 업적, 모두 "숫자 도달" 타입
"score_1000": { "name": "천 점 돌파", "threshold": 1000 }
"combo_3": { "name": "콤보 입문", "threshold": 3 }
"games_100": { "name": "중독자", "threshold": 100 }
```

**문제:**
- 100판 해야 달성하는 업적이 "중독자"라니... 게임이 스스로 중독이 안 된다는 걸 인정하는 꼴
- 모든 업적이 **누적형**(play more = get it). **기술형** 업적이 없음
- 업적 달성해도 보상이 없음 — 그냥 체크마크
- 업적 진행률이 게임 중 보이지 않음 → 존재감 제로

#### D. 데일리 챌린지 — "같은 게임 + 시드 고정"

`daily_challenge_system.gd` 분석:

```gdscript
# 오늘 날짜 기반 시드 → 같은 퍼즐
# 차이점: 시드만 고정. 게임 규칙 동일. 특별한 목표 없음.
```

**문제:** 데일리 챌린지라고 하지만 그냥 "시드가 고정된 같은 게임"이다. 목표 점수도 없고, 다른 플레이어와 비교도 안 되고, 클리어 보상도 없다.

#### E. 데일리 보상 — "사실상 미사용"

`daily_reward_system.gd`에 7일 사이클 보상이 정의되어 있지만:
- `score_multiplier`와 `bonus_swaps`가 실제 게임 로직에 연결된 흔적이 없음
- `check_in()`을 호출하는 코드가 메인 게임 루프에 없음
- **구현만 되어 있고 활성화되지 않은 시스템**

### 3.3 Variable Reward 분석 (핵심)

Nir Eyal의 《Hooked》 모델에서 중독적 제품의 핵심은 **Variable Reward (변동 보상)**이다.

#### ChromaBlocks의 변동성 점검

| 보상 요소 | 예측 가능성 | 변동성 | 도파민 유발 |
|----------|-----------|--------|-----------|
| 점수 | 완전히 예측 가능 (룰 기반) | ❌ 없음 | 최소 |
| 콤보 | 반 예측 가능 | ⚠️ 약간 | 중간 |
| Chroma Chain | 반 예측 가능 (5+셀 인접) | ⚠️ 약간 | 중간 |
| Chroma Blast | 반 예측 가능 (6+셀 같은색) | ⚠️ 약간 | 중간 |
| 퍼펙트 클리어 | 극히 드물지만 예측 가능 | ⚠️ 약간 | 높음 (드물어서) |
| **럭키 드롭 / 서프라이즈** | — | ❌ **없음** | ❌ **없음** |

**결론:** 핵심 보상 메커니즘 중 "완전히 예측 불가능한" 보상이 **단 하나도 없다.** Chroma Chain과 Blast는 숙련 플레이어에겐 계산 가능한 결과이지 "서프라이즈"가 아니다.

#### Block Blast와의 변동 보상 비교

Block Blast는 같은 블록 퍼즐인데 왜 중독적인가?
- **점수 폭발 순간:** 여러 줄 동시 클리어 시 점수가 기하급수적으로 올라가는 느낌
- **보드 상태의 변동성:** 좋아 보이다가 갑자기 위기, 위기에서 극적 탈출
- **"아 이거 놓으면 3줄 간다!" 기대감:** 핵심은 보상 자체가 아니라 **보상 기대**

## 4. 감정 곡선 분석

### 4.1 ChromaBlocks 감정 타임라인

```
감정
흥분  ↑
     │   ① 시작        ③ 첫 콤보     ⑤ 위기 탈출?
     │   ┌─┐          ┌──┐          ┌─┐
     │   │ │    ②     │  │   ④     │ │    ⑥ 게임오버
평온 │───┘ └──────────┘  └─────────┘ └──────────┐
     │         (긴 평탄 구간)    (또 평탄)          │
지루 │    ↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓↓         │
     │                                           └──→
좌절 │                                            ↓
     └──────────────────────────────────────────────→ 시간
```

### 4.2 구간별 분석

#### ① 게임 시작 (0-30초)
- **보드 30% 프리필** — 좋은 디자인. 빈 보드보다 즉시 "할 게 있다" 느낌
- 첫 피스 배치 → 작은 만족감
- **문제:** 프리필 블록이 랜덤이라 첫 클리어까지 걸리는 시간이 천차만별

#### ② 빌드업 (30초-3분) — ⚠️ **위험 구간**
- 피스를 놓고, 놓고, 놓고... 클리어가 안 됨
- 보드가 서서히 차오르지만 "완성"까지가 너무 오래 걸림
- **감정: 무감각. 기계적 반복.** 이 구간에서 대부분이 이탈함
- `_prefill_board`의 30% 프리필이 색상 클러스터를 만들지만, 라인 완성과는 무관한 랜덤 배치

#### ③ 첫 클리어/콤보 (간헐적)
- 1줄 클리어 → 100점 + 이펙트 → 작은 쾌감
- 2줄+ → "Nice!" + 파티클 → 중간 쾌감
- **문제:** 쾌감 피크 사이의 간격이 너무 김. 10초에 한 번은 "뭔가 일어나야" 하는데, 30초-1분 동안 아무 일도 안 일어날 수 있음

#### ④ 중반 (3-8분)
- 보드가 50-70% 차오름
- 전략적 사고가 필요해지지만... "왜 이걸 잘 해야 하지?"
- **감정: 의무감.** 재미가 아니라 "시작했으니까" 하는 느낌

#### ⑤ 위기/탈출 (간헐적)
- 보드 80%+ → Crisis 표시 (edge vining)
- 극적인 라인 클리어로 탈출 → 가장 큰 쾌감 피크
- **문제:** 이 순간이 너무 드물고, 탈출해도 "그래서 뭐?" 느낌

#### ⑥ 게임오버
- "GAME OVER" → Grade (S/A/B/C) → 점수 카운트업 → 스탯
- **문제:** 아래 "한판만 더" 분석에서 자세히

### 4.3 쾌감 피크 빈도 분석

**이상적 쾌감 피크 빈도:** 8-12초에 한 번 (모바일 캐주얼 기준)
**ChromaBlocks 실제:** 20-60초에 한 번 (클리어 간격 기준)

8x8 보드에서 라인 하나를 완성하려면 8칸을 채워야 한다. 피스 평균 크기가 3-4셀이라 하면, 한 줄을 채우는 데 최소 2-3턴이 필요하다. 하지만 실제로는 여러 줄에 걸쳐 동시에 채워가므로 첫 클리어까지 5-8턴, 즉 20-40초가 걸린다.

**Block Blast와의 차이:**
Block Blast도 비슷한 구조지만, **스코어 팝업이 매 배치마다 나옴** + **작은 보상(배치 자체의 피드백)이 더 쥬시**하다. ChromaBlocks는 배치 자체의 피드백이 있지만(`play_place_effect`), 숫자적 보상(5점/셀)이 너무 미미해서 체감이 안 됨.

## 5. 이탈 포인트 분석

### 5.1 이탈 시점 진단

```
이탈 확률
    ↑
100%│                                        ● 게임오버
    │                                       ╱  (자연 이탈)
 75%│                            ●─────────╱
    │                           ╱
 50%│          ●───────────────╱ ← "빌드업 이탈"
    │         ╱                    (지루해서 나감)
 25%│   ●────╱
    │  ╱ 
  0%│─╱──────────────────────────────────────→
    0   1min   3min   5min   8min   GameOver
```

### 5.2 이탈 원인: 좌절감 vs 지루함

**결론: 지루함이 주된 원인이다.**

| 분석 요소 | 좌절감 | 지루함 |
|----------|--------|--------|
| "놓을 곳이 없다!" (게임오버 직전) | ✅ 있음 | — |
| "뭘 해야 할지 모르겠다" (중반) | — | ✅ 주범 |
| "계속 같은 일의 반복" | — | ✅ 주범 |
| "이겨도 별 감흥이 없다" | — | ✅ 주범 |
| "어렵지만 하고 싶다" | ❌ 발생 안 함 | — |

**좌절감은 있지만 건강한 좌절감이 아니다.** 건강한 좌절감은 "아 이것만 넘기면 되는데!" (=다시 도전 욕구). ChromaBlocks의 좌절감은 "어쩔 수 없이 끝남" (=체념).

### 5.3 코드 기반 근거

`game_over_system.gd`:
```gdscript
static func is_game_over(board, tray_pieces, held_piece, hold_used) -> bool:
    # 남은 모든 피스를 놓을 수 없으면 → 게임오버
    return not board.can_place_any_piece(all_pieces)
```

게임오버는 "가능한 수가 없을 때" 발생한다. 이것은 기계적이고 건조하다. 플레이어가 **"아 여기에 이걸 놓았어야 했는데!"** 라고 느끼기엔, 게임오버의 원인이 너무 분산적이다 (어디서 실수했는지 알기 어려움).

## 6. "한판만 더" 트리거 분석

### 6.1 게임오버 순간의 감정 설계

게임오버 화면 (`game_over_screen.gd`) 분석:

**현재 제공하는 정보:**
1. Grade (S/A/B/C) — 점수 기반
2. 점수 카운트업 애니메이션
3. 최고점수 비교
4. 스탯 4개: Lines, Blocks, Combos, Chains
5. 미션 결과 (미션 모드일 때)
6. "PLAY AGAIN" 버튼

**좋은 점:**
- Grade 시스템으로 "내가 얼마나 잘 했는지" 즉시 파악
- 점수 카운트업 애니메이션으로 서스펜스
- 이번 판 하이라이트 스탯

**결정적 문제점:**
- **Near-miss 효과가 전혀 없다**
- **"다음엔 이걸 해봐" 같은 목표 제안이 없다**
- **이전 판과의 비교가 약하다**

### 6.2 Near-Miss 효과 부재

Near-miss 효과란: 슬롯머신에서 ♦♦♣ 가 나왔을 때 "아 하나만 더 맞았으면!" 느끼는 것. 이것이 다시 돌리게 만든다.

**ChromaBlocks에서 Near-miss가 작동하지 않는 이유:**

1. **점수 기반 Grade 임계값이 너무 넓다:**
```gdscript
const GRADE_THRESHOLDS := { "S": 10000, "A": 5000, "B": 2000 }
```
- B등급은 2000-4999 사이. 범위가 3000점이나 된다.
- 2100점으로 B를 받으면 "아 조금만 더 하면 A였는데"가 아니라 "B네. 그래."
- **등급 사이 간격이 너무 넓어서 "아까움"이 발생하지 않음**

2. **게임오버 시 "가능했던 수"를 보여주지 않음:**
- 체스의 "이 수를 놓쳤습니다" 같은 피드백이 없음
- 플레이어는 자신의 실수를 인식하지 못함 → 개선 동기 없음

3. **이전 판과의 비교가 너무 약함:**
- "Best: 25,100" 만 작게 표시
- "이전 판보다 +1,200점 더 했어요!" 같은 비교 없음
- 진보하고 있다는 느낌을 주지 않음

### 6.3 "한판만 더" 트리거 체크리스트

| 트리거 | Block Blast | ChromaBlocks | 상태 |
|--------|------------|-------------|------|
| "아 조금만 더 하면 됐는데!" (Near-miss) | ✅ 높은 점수 + 랭킹 비교 | ❌ 넓은 Grade 구간 | 미작동 |
| "이번엔 이 전략 써봐야지" (학습) | ✅ 단순 규칙 → 전략 깊이 | ⚠️ 전략은 있지만 피드백 부족 | 약함 |
| "조금만 더 하면 보상!" (진행률) | ✅ 레벨 진행 바 | ⚠️ 레벨업 의미 부족 | 약함 |
| "친구가 나보다 잘함" (사회적) | ✅ 리더보드 | ❌ 없음 | 미작동 |
| "데일리 안 하면 손해" (손실 회피) | ✅ 데일리 보상 | ❌ 실질 보상 없음 | 미작동 |
| "이 업적만 깨면 되는데" (수집) | ⚠️ 약함 | ❌ 업적 보상 없음 | 미작동 |

**6개 중 0개 작동.** 이것이 "한판만 더" 욕구가 안 생기는 근본 원인이다.

## 7. Block Blast와의 비교 분석

### 7.1 같은 장르, 다른 결과

두 게임의 핵심 메커니즘은 거의 동일하다: 8x8 보드, 3피스 트레이, 드래그앤드롭, 줄 클리어. 그런데 Block Blast는 중독적이고 ChromaBlocks는 아니다. 왜?

### 7.2 감정 곡선 비교

```
Block Blast:
흥분 │   ╱╲   ╱╲  ╱╲╱╲  ╱╲╱╲╱╲  ← 점점 빈도와 강도 증가
     │  ╱  ╲ ╱  ╲╱    ╲╱      ╲
평온 │─╱────╲╱──────────────────╲──→
     │                           ↓ 게임오버 (아까워!)

ChromaBlocks:
흥분 │   ┌┐       ┌┐      ┌┐     ← 빈도 낮고 강도 일정
     │   ││       ││      ││
평온 │───┘└───────┘└──────┘└──────→
     │                           ↓ 게임오버 (그래.)
```

### 7.3 핵심 차이 분석

| 요소 | Block Blast | ChromaBlocks | 영향 |
|------|------------|-------------|------|
| **보상 밀도** | 배치마다 체감되는 진전 | 클리어 때만 체감 | BB가 3배 이상 높은 보상 빈도 |
| **점수 곡선** | 기하급수적 성장감 | 선형 성장 | BB가 "우와 점수 폭발" 느낌 |
| **난이도 곡선** | 점진적, 매 순간 긴장 | 빌드업→갑자기 어려움 | BB가 플로우 존 더 오래 유지 |
| **게임오버 감정** | "아 거의 다 했는데!" | "끝났네" | BB의 Near-miss 효과 작동 |
| **세션 길이** | 3-5분 (빠른 루프) | 5-15분 (긴 루프) | BB의 빠른 재도전 유도 |

### 7.4 ChromaBlocks만의 차별점 (강점)

하지만 ChromaBlocks에는 Block Blast에 없는 것도 있다:
- **Chroma Chain:** 연쇄 클리어의 전략적 깊이 → 숙련 보상
- **Chroma Blast:** 색상 테마 보너스 → 전략적 색 배치
- **Hold 시스템:** 전략적 선택지 확대
- **미션 모드:** 구조적 목표 제공
- **Daily Challenge:** 시드 기반 공정 경쟁

이것들은 **좋은 시스템**이지만, 감정적 보상이 충분히 설계되지 않아서 **체감이 안 되는 것**이 문제다.

## 8. 실행 가능한 개선안

### 🔴 Critical (즉시 적용 — 중독성 핵심)

#### 8.1 Variable Reward 시스템 도입

**이론:** Skinner의 Variable Ratio Schedule은 Fixed Ratio 대비 4-6배 높은 반응률을 보인다.

**구현안: "Lucky Clear" 시스템**

```
발동 조건: 라인 클리어 시 15% 확률
보상:
  - Lv1 (10%): 점수 1.5배 + 황금 파티클
  - Lv2 (4%):  점수 2배 + 무지개 파티클 + 특별 사운드  
  - Lv3 (1%):  점수 3배 + 전체화면 이펙트 + "LUCKY!" 팝업
```

**왜 이것이 작동하는가:**
- 매 클리어마다 "이번엔 Lucky일까?" 기대감
- 예측 불가능 → 도파민 시스템 활성화
- 추가 코드 복잡도 낮음 (점수에 배율 적용 + 이펙트 분기)

**코드 스케치:**
```gdscript
# scoring_system.gd에 추가
var lucky_roll := randf()
var lucky_mult := 1.0
var lucky_tier := 0
if lucky_roll < 0.01:
    lucky_mult = 3.0; lucky_tier = 3
elif lucky_roll < 0.05:
    lucky_mult = 2.0; lucky_tier = 2
elif lucky_roll < 0.15:
    lucky_mult = 1.5; lucky_tier = 1
```

#### 8.2 Near-Miss Grade 시스템

**현재:** `{ "S": 10000, "A": 5000, "B": 2000 }`
**개선:** 등급 세분화 + "아까움" 표시

```gdscript
# 등급 세분화
const GRADE_THRESHOLDS := {
    "S+": 15000, "S": 10000,
    "A+": 7500,  "A": 5000,
    "B+": 3500,  "B": 2000,
    "C+": 1000,  "C": 0,
}

# 게임오버 화면에서:
# "A등급까지 320점만 더 있으면 됐어요!"
# → 진행률 바로 표시 (현재 점수 / 다음 등급)
# → Near-miss 효과 극대화
```

#### 8.3 "이번 판" → "다음 판" 연결

**게임오버 화면 개선:**

현재: Grade + 점수 + 스탯 → "PLAY AGAIN"

개선:
```
GAME OVER
━━━━━━━━━━━━━━━━━━━
Grade: A (4,800점)
━━━━━━━━━━━━━━━━━━━
🏆 이전 판보다 +1,200점!     ← 진보 피드백
📊 A+ 까지 700점만 더!       ← Near-miss
🔥 최대 콤보: 5 (역대 최고!) ← 개인 기록

💡 이번 판 발견:
   "Chroma Chain 2회 발동 — 색 배치를 의식하면 더 자주 발동됩니다"
   ← 학습 + 다음 판 전략 힌트

[▶ 한판만 더]    [🏠 홈]
```

### 🟠 High (1주 내 적용 — 목표 구조)

#### 8.4 마이크로 목표 시스템

**이론:** Locke의 목표 설정 이론 — 구체적이고 도전적인 목표가 모호한 목표보다 25-35% 높은 성과를 만든다.

**구현안: 매 게임 시작 시 3개 미니 목표**

```
이번 판 목표:
☐ 2줄 동시 클리어 1회    (+200 보너스)
☐ 콤보 x3 달성           (+300 보너스)  
☐ 1,500점 달성           (+500 보너스)
```

이것은 이미 `mission_system.gd`에 구현되어 있다! 하지만 **미션 모드에서만** 작동한다. 이것을 **모든 게임에 기본 적용**하면 매 판마다 "할 일"이 생긴다.

**코드 변경 범위:**
- `_start_new_game()`에서 항상 `MissionSystem.generate_missions()` 호출
- `_mission_hud`를 항상 표시
- 미션 완료 시 보너스 점수 즉시 지급

#### 8.5 레벨업 의미 부여

현재 레벨업은 레벨 25/30까지 **아무 효과가 없다.** 

**개선안:**

| 레벨 | 변화 |
|------|------|
| 1-3 | 기본 (4색, 쉬운 피스) |
| 4-5 | +1색 (5색) — "새 색상 해금!" 이벤트 |
| 6-8 | 피스 복잡도 증가 — 이미 구현됨 |
| 9-10 | +1색 (6색) — "새 색상 해금!" |
| 11-15 | Chain 임계값 4로 감소 → 체인 빈도 증가 |
| 16-20 | Blast 임계값 5로 감소 → 블라스트 빈도 증가 |
| 21-25 | 특수 타일 활성화 (현재 비활성) |
| 26+ | 전문가 모드 (현재 로직 유지) |

핵심: **레벨업이 "새로운 것이 열리는" 순간이 되어야 한다.**

#### 8.6 데일리 챌린지 의미 부여

**현재:** 시드만 고정. 목표 없음.
**개선:**

```
오늘의 챌린지
━━━━━━━━━━━━━━
🎯 목표 점수: 5,000
⭐ 별 획득 조건:
   ★☆☆ 3,000점 달성
   ★★☆ 5,000점 달성
   ★★★ 8,000점 달성
   
🔥 연속 출석: 5일째 (7일 보상까지 2일!)
📊 전체 플레이어 상위 32% (추정)
```

### 🟡 Medium (2주 내 — 감정 강화)

#### 8.7 쾌감 피크 빈도 증가

**문제:** 클리어 간 간격이 너무 길다 (20-60초).
**목표:** 8-12초마다 "뭔가 일어나는" 느낌.

**구현안: "프로그레스 포인트"**

보드에 특정 패턴이 형성되면 (아직 완성은 아니지만) 시각적 힌트:
- 라인 7/8 완성 → 해당 줄 미세한 글로우 (현재 `show_line_prediction`은 드래그 중에만 작동)
- 같은색 4셀 인접 → Chroma Chain 힌트 미세한 연결선
- 이런 "거의 다 됐다!" 시각적 피드백이 중간중간 보상 역할

#### 8.8 점수 폭발 구간 설계

**현재:** 1줄=100, 2줄=300, 3줄=600, 4줄=1000 → 선형에 가까운 성장

**개선:** 비선형 보상 곡선
```gdscript
# 현재
const LINE_CLEAR_POINTS: Dictionary = {1: 100, 2: 300, 3: 600, 4: 1000}

# 개선안: 멀티라인 보상 폭발적 증가
const LINE_CLEAR_POINTS: Dictionary = {1: 100, 2: 350, 3: 800, 4: 1500}
# + 5줄 이상: 1500 + (lines-4) * 800 (현재: 1000 + (lines-4) * 500)
```

핵심은 "대박 클리어"가 정말로 **대박처럼 느껴지는** 것이다.

#### 8.9 사운드 계이름 피드백 (Tetris Effect 영감)

`ux-psychology-research.md`에 이미 언급된 아이디어:

> "블록을 놓을 때마다 음계가 올라감 (도→레→미...). 연속 배치가 멜로디가 됨."

이것은 배치 자체에 **감각적 보상**을 부여한다. 현재 `sound_manager.gd`에서 `play_sfx("block_place")`가 항상 같은 소리를 내는 것 대신, 배치 순서에 따라 음이 올라가면 "리듬감" + "연속성" + "진전감"이 모두 생긴다.

### 🟢 Low (여유 있을 때 — 장기 리텐션)

#### 8.10 소셜 비교 (가벼운 버전)

리더보드까지 갈 필요 없이:
- 데일리 챌린지 점수를 디바이스 로컬에 기록
- "상위 N%에 해당합니다" (실제 서버 없이, 예상 분포로 추정)
- 심지어 허구의 비교라도 Near-miss 효과를 발생시킴

#### 8.11 업적 보상 연결

현재 업적은 달성해도 아무 보상이 없다. 
- 업적 달성 → 즉시 점수 보너스 or 특별 피스 1회 지급
- "중독자 (100판)" 같은 이름 대신 긍정적 이름: "마스터 플레이어"
- 업적 진행률을 게임 중 HUD에 미세하게 표시

#### 8.12 "끝나기 직전" 드라마 연출

보드 80%+ 일 때:
- BGM 템포 증가 (이미 `_update_bgm_intensity`에서 intensity 2 적용)
- 하지만 여기에 추가: **"DANGER" 표시 + 타이머 없는 긴장감 UI**
- 이 구간에서 클리어하면 **보너스 점수 1.5배** → "위기 탈출 보상"
- 이것이 "아 거의 죽을 뻔했는데 살았어!" → "한판만 더" 트리거
