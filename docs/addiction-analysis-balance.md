# ChromaBlocks 중독성 분석 — Balance & Feel 관점

> 분석일: 2026-03-25 | 분석 범위: 점수 피드백, 이펙트/주스, 피스 생성 밸런스, 게임오버 타이밍, 속도감

## 1. 점수 피드백 분석

### 1.1 점수 공식 (scoring_system.gd + game_constants.gd)

| 행동 | 점수 | 비고 |
|------|------|------|
| 피스 배치 | 셀당 5pt | 예: 4셀 테트로미노 = 20pt |
| 1라인 클리어 | 100pt | |
| 2라인 클리어 | 300pt | 1.5배 아닌 3배 점프 ✅ |
| 3라인 클리어 | 600pt | |
| 4라인 클리어 | 1,000pt | |
| 5+라인 | 1,000 + (n-4)×500pt | |
| 퍼펙트 클리어 | 2,000pt 보너스 | |

**콤보 배수:**

| 콤보 | x1 | x2 | x3 | x4 | x5 | x6 | x7 | x8+ |
|------|-----|-----|-----|-----|-----|-----|-----|------|
| 배수 | 1.0 | 1.2 | 1.5 | 2.0 | 2.5 | 3.0 | 3.5 | 4.0 |

**듀얼 클리어 보너스:** 라인 클리어 + 컬러 매치 동시 발생 시 ×1.25 추가 배수

**Chroma Chain:** 5+ 동색 인접 셀 연쇄 제거. 캐스케이드별 셀당 30/60/90pt  
**Chroma Blast:** 클리어 라인에 동색 6+ 셀 시 해당 색 전체 보드에서 제거. 셀당 50pt + 트리거당 500pt

### 1.2 도파민 히트 분석

**현재 점수 곡선:**
```
1라인:  100pt → 콤보x3이면 150pt → 콤보x5이면 250pt
2라인:  300pt → 콤보x3이면 450pt → 콤보x5이면 750pt
3라인:  600pt → 콤보x3이면 900pt → 콤보x5이면 1,500pt
```

**🟢 잘 된 부분:**
- 2라인→3라인 점프가 2x (100→300→600)로 **비선형 보상** — 멀티클리어 유인 효과 있음
- 콤보 배수가 1.0→1.2→1.5→2.0으로 **가속적** — x4 이후부터 "폭발" 느낌
- Chroma Blast + Chain의 연쇄 점수가 수천점 — **잭팟 도파민** 제공

**🔴 문제점:**
- **배치 점수(셀당 5pt)가 너무 미미함.** 총점 대비 1-3% 수준. 배치 자체에 보상감 없음
- 1라인 클리어 100pt → 2라인 300pt 점프는 좋지만, **1라인 자체가 너무 시시함** (배치 20pt + 클리어 100pt = 120pt)
- 콤보 x1~x2 구간(1.0→1.2)의 차이가 **거의 체감 불가** — 20% 증가는 시각적으로 안 보임
- **Color Match가 비활성화** (`COLOR_MATCH_ENABLED = false`) — 6셀 매치 200pt, 8셀 500pt의 보너스 기회가 사라져 있음

### 1.3 Block Blast 비교: "POW!" 느낌

Block Blast의 핵심: **라인 클리어 순간의 폭발적 피드백** + **연쇄 클리어의 카타르시스**

| 요소 | Block Blast | ChromaBlocks | 차이 |
|------|-------------|--------------|------|
| 1라인 클리어 점수 | ~10pt 기본 | 100pt | CB가 기본값 높음 |
| 콤보 증폭 | 누적형 (기하급수) | 배수형 (최대 4x) | BB의 무한 스케일링이 더 중독적 |
| 스코어 팝업 | 크고 화려한 숫자 | 작은 +숫자 라벨 | **CB 약함** |
| 멀티클리어 피드백 | 화면 전체 이펙트 | "DOUBLE/TRIPLE" 텍스트 | 유사 |

### 1.4 개선 제안

| 항목 | 현재 | 제안 | 이유 |
|------|------|------|------|
| 배치 점수 | 5pt/셀 | **10pt/셀** | 배치 자체에 미니 보상감 부여 |
| 1라인 클리어 | 100pt | **150pt** | 기본 클리어의 만족감 향상 |
| 콤보 x2 배수 | 1.2 | **1.4** | x1→x2 전환 시 체감 차이 확보 |
| 스코어 팝업 사이즈 | 22px~32px | **28px~40px** | 시각적 임팩트 강화 |
| Color Match | 비활성 | **활성화 권장** | 색상 전략 + 추가 도파민 경로 |

## 2. 이펙트/주스(Juice) 분석

### 2.1 피스 배치 이펙트 (cell_view.gd: `play_place_pulse`)

**시퀀스:**
1. 색상 플래시 → 밝은 색으로 변경 + scale 1.05x (즉시)
2. **Squash:** 1.05 → 0.94 (60ms, EASE_OUT)
3. **Stretch:** 0.94 → 1.06 (50ms, EASE_OUT + TRANS_BACK)
4. **Settle:** 1.06 → 1.0 (40ms, EASE_IN_OUT)
5. 색상 페이드백 150ms (병렬)

**총 시간: ~150ms** (delay 포함 시 셀당 +15ms 스태거)

**인접 리플:** 배치 셀 주변 2칸 이내 점유 셀이 1.02x 미세 바운스 (50ms+50ms)

**보드 바운스:** `play_place_bounce()` — Y축 +2px (40ms) → 0 (80ms, TRANS_BACK)

**🟢 양호:** squash-stretch 원리 적용, 웨이브 스태거, 인접 리플 — **물리적 "착지" 느낌 있음**
**🟡 아쉬움:** 스케일 변화량(±6%)이 미세해서 작은 화면에서 **거의 안 보일 수 있음**

### 2.2 라인 클리어 이펙트 (cell_view.gd: `play_clear_flash`)

**시퀀스:**
1. **Bright Flash:** 셀 색상 → 밝은 색 + 글로우 풀 (즉시)
2. **Peak Pop:** +60ms → 40% 밝게 + scale 1.2x
3. **Scale Up:** 1.0→1.2 (80ms, TRANS_BACK)
4. **Shrink + Fade:** 1.2→0.0 + 색상→빈 셀 (200ms, EASE_IN + TRANS_QUAD)
5. 리셋

**총 시간: ~340ms** (셀당 +20ms 스태거)

**파티클 시스템 (clear_particles.gd):**
- 최대 48 파티클 (intensity 2.0에서 64)
- 5가지 셰이프: RECT, DIAMOND, CIRCLE, STAR, SPARK
- 잔상(afterglow trail): 이전 위치에 ghost 렌더링
- 스크린 플래시: intensity ≥1.5 시 0.15~0.35 알파 화면 플래시
- 2차 폭발: intensity ≥1.5 시 120ms 후 미니 폭발 추가
- 수명: 1.0초, 중력 240px/s², 0.4초 이후 페이드 시작

**쇼크웨이브 링:**
- 중심에서 확장하는 원형 링 (0.4초, 최대 반경 cell_size × 4)
- 1.5+ 강도: 2차 링 (1.6× 반경, 0.4 알파)
- 2.0+ 강도: 3차 화이트 링 (2.0× 반경, 0.3 알파)

**스크린 셰이크 (board_renderer.gd):**

| 라인 수 | 강도(px) | 지속시간 | 오실레이션 |
|---------|---------|---------|----------|
| 1 | 3px | 80ms | 6회 |
| 2 | 5px | 120ms | 6회 |
| 3+ | 8px | 180ms | 6회 |

### 2.4 사운드 디자인 (sfx_generator.gd + sound_manager.gd)

**생성 방식:** 런타임 합성 (8비트 레트로 사운드) — 에셋 파일 없음, 백그라운드 스레드에서 생성

**볼륨 매트릭스 (dB):**

| 사운드 | 볼륨 | 빈도 | 평가 |
|--------|------|------|------|
| block_place | -14dB | 매우 높음 | ✅ 적절히 조용 |
| line_clear (1) | -8dB | 높음 | ✅ |
| line_clear (4) | -3dB | 낮음 | ✅ 보상적 |
| combo x1 | -8dB | 높음 | ✅ |
| combo x5+ | -3dB | 낮음 | ✅ 클라이맥스 |
| place_fail | -10dB | 중간 | ✅ |
| perfect_clear | -3dB | 매우 낮음 | ✅ 특별함 |

**🟢 볼륨 밸런스 우수:** 빈도↑ → 볼륨↓, 보상↑ → 볼륨↑ 원칙 충실

**라인 클리어 SFX 특성:**
- 기본: 400Hz→2400Hz 상승 스윕 (250ms)
- 라인 수↑: 시작 주파수 +200Hz/라인, 도착 주파수 +500Hz/라인
- **더 많은 라인 = 더 높고 화려한 사운드** ✅

**콤보 SFX 특성:**
- x1: C5 단음 (80ms)
- x2: C5→E5 이중음 (150ms)
- x3: C5→E5→G5 삼중 아르페지오 (200ms)
- x4+: 풀 런 + 스파클 (300ms)
- **점진적 복잡도 증가** ✅

### 2.6 이펙트 종합 평가

**🟢 강점:**
- 멀티레이어 피드백: 시각(파티클+쇼크웨이브+스크린셰이크) + 청각(합성 SFX) + 촉각(햅틱)이 동기화
- Hit Stop (Engine.time_scale=0): 라인 클리어 시 20~60ms 프레임 정지 — **임팩트 느낌 강화**
- wall-clock 타이밍으로 hit-stop 중에도 팝업 애니메이션이 정상 진행
- 카메라 줌 펄스 기능 존재 (`play_camera_zoom`)

**🔴 부족한 부분:**
- **Score Cascade(점수 분해 팝업)가 코드에 존재하지만 `_play_effects_sequence`에서 호출되지 않음** — `_spawn_score_cascade(ed)` 미사용
- **Anticipation Phase(클리어 전 긴장감)가 코드에 있지만 `_play_effects_sequence`에서 호출되지 않음** — `play_clear_anticipation` 미사용
- 카메라 줌 펄스도 이펙트 시퀀스에서 호출 안 됨
- Near-miss 힌트 업데이트가 **주석 처리됨** (`# board_renderer.update_near_miss_hints(board)`)
- **배치 시 리플/바운스가 클리어 없을 때만** 발생 — 클리어 시에도 배치 느낌은 필요함

### 2.7 이펙트 개선 제안

| 항목 | 현재 | 제안 | 기대 효과 |
|------|------|------|----------|
| Anticipation Phase | 미사용 | 라인 클리어 직전 100ms 딤+펄스 활성화 | "곧 터진다!" 기대감 |
| Score Cascade | 미사용 | 3+라인 또는 Blast 발생 시 활성화 | 점수 분해 쾌감 |
| Camera Zoom | 미사용 | 3+라인 시 1.03x 줌 펄스 (200ms) | 화면 집중 효과 |
| Near-miss 힌트 | 주석 처리 | 재활성화 | "거의 완성!" 긴장감 |
| 배치 스케일 | ±6% | **±10%** | 작은 화면에서도 체감 |
| Place Bounce | 2px | **4px** | 배치 착지감 강화 |
| 클리어 시 배치 바운스 | 없음 | 클리어 여부 무관하게 바운스 | 항상 배치 느낌 |

## 3. 피스 생성 밸런스

### 3.1 피스 카테고리

| 카테고리 | 피스 수 | 셀 수 | 예시 |
|---------|--------|-------|------|
| TINY | 3 | 1~2 | SINGLE, DUO, DUO_V |
| SMALL | 4 | 3 | TRI_LINE, TRI_L 등 |
| MEDIUM | 15 | 4 | 테트로미노 전종 (T, Z, S, L, J, I, O 변형) |
| LARGE | 10 | 5 | 펜토미노 (PLUS, U, T, L3, N 등) |
| HUGE | 3 | 6~9 | RECT_2x3, SQ_3x3, RECT_3x2 |

**총 35종의 피스** — 다양성 ✅

### 3.2 DDA (Dynamic Difficulty Adjustment) 시스템

```
보드 밀도 < 20%  →  RUSH 모드 (50% 확률): 대형 피스 위주
보드 밀도 < 40%  →  일반 템플릿
보드 밀도 ≥ 40%  →  MILD 자비: 작은 피스 포함 템플릿 가중치 1.8x
보드 밀도 ≥ 60%  →  STRONG 자비: 작은 피스 포함 3.0x, 없으면 0.3x
보드 밀도 ≥ 75%  →  CRITICAL 자비: 첫 피스 무조건 SMALL 이하 보장
```

**FIT 체크 (배치 가능 피스 필터):**
- 밀도 ≥ 40%: 70% 확률로 배치 가능한 피스만 풀에서 선택
- 밀도 ≥ 75%: 90% 확률

**Rescue 시스템 (3단계):**
1. **DOWNSIZE:** 가장 큰 미배치 피스 → 한 단계 작은 카테고리로 교체
2. **FIT_SCAN:** TINY/SMALL/MEDIUM에서 아무 배치 가능 피스 탐색
3. **SINGLE_FORCE:** 빈 셀 있으면 1×1 단일 피스 강제 삽입

**Relief 트레이:** 연속 2회 tight tray(배치 가능 1개 이하) 후 → TINY + SMALL + SMALL 안전 트레이

### 3.3 레벨별 템플릿 분포

**Level 1~5 (Easy):**
```
22%: SMALL + MEDIUM + MEDIUM    ← 가장 흔한 조합
15%: MEDIUM × 3
13%: TINY + MEDIUM + MEDIUM
12%: SMALL + MEDIUM + LARGE
...
```

**Level 6~15 (Medium):**
```
25%: SMALL + MEDIUM + MEDIUM
20%: TINY + MEDIUM + LARGE
15%: SMALL + SMALL + LARGE
...
```

**Level 30+ (Expert):** 트레이 2개로 축소!
```
30%: MEDIUM + LARGE
25%: SMALL + LARGE
20%: LARGE + LARGE
15%: MEDIUM + HUGE
10%: SMALL + HUGE
```

### 3.4 "딱 맞는 피스" 빈도 분석

**색상 자비 시스템:**
- 보드 밀도 >35%이면 40% 확률로 가장 큰 동색 클러스터(4+셀)의 색상 피스 생성
- 기본 색상 클러스터링: 25% 확률로 트레이 내 첫 피스와 동색

**피스 반복 방지:**
- 이전 트레이와 같은 타입 → 가중치 0.4x (60% 감소)

**피스 점진적 해금:**
- Lv1~2: 복잡한 펜토미노 6종 제외
- Lv3~4: PLUS, U만 제외
- Lv5~7: PLUS만 제외
- Lv8+: 전종 해금

### 3.5 밸런스 평가

**🟢 강점:**
- DDA가 Block Blast 스타일의 "빈 보드→큰 피스, 밀집→작은 피스" 메커니즘 충실
- 3단계 Rescue + Relief 트레이로 불공정 게임오버 방지
- 색상 자비가 Chroma Chain(5셀 연쇄)에 연결 — "주면서 빼기" 구조

**🔴 문제점:**
- **RUSH 모드(빈 보드) 확률이 50%로 너무 낮음.** 빈 보드에서 작은 피스가 나오면 지루함
- **TINY 피스(1~2셀)가 너무 단순해서 만족감 없음.** 배치 점수도 5~10pt뿐
- **HUGE 카테고리가 3종(2×3, 3×3, 3×2)뿐** — 8×8 보드에서 너무 큰 감이 있음
- Expert 모드(Lv30+)에서 트레이 2개 + 큰 피스 = **급격한 난이도 절벽**

### 3.6 개선 제안

| 항목 | 현재 | 제안 | 이유 |
|------|------|------|------|
| RUSH 확률 | 50% | **70%** | 빈 보드 = 공격 기회, 더 적극적으로 |
| TINY 배치 보너스 | 없음 | **TINY 배치 시 +20pt 추가** | 작은 피스 선택의 전략적 가치 |
| SQ_3x3 (9셀) | HUGE에 포함 | **별도 "EPIC" 카테고리** | 9셀 피스는 너무 극단적, 출현율 분리 관리 |
| Expert 트레이 | 2개 | **3개 (SMALL 추가)** | 난이도 절벽 완화 |
| 색상 자비 활성 밀도 | 35% | **25%** | 초반부터 색상 전략 유도 |

## 4. 게임오버 타이밍

### 4.1 위기 경고 시스템 (board_renderer.gd)

| 밀도 | 보더 색상 | 펄스 주기 | 평가 |
|------|----------|----------|------|
| <70% | 기본 (#3D3D5C) | 없음 | ✅ |
| 70~80% | AMBER (0.8α) | 600ms | ✅ 경고 |
| 80%+ | CORAL | 300ms | ✅ 위기 |

**기존 Near-miss 힌트 (비활성):**
- 7/8 채워진 행/열에 흰색 펄스 (1.5초 주기, α 0.10~0.20)
- 4+ 동색 클러스터에 색상 글로우 (1.5초 주기, α 0.08~0.16)

### 4.2 Mercy 시스템

```
밀도 40%+ → 작은 피스 템플릿 가중치 1.8x (MILD)
밀도 60%+ → 작은 피스 3.0x, 큰 피스 0.3x + FIT 체크 70% (STRONG)
밀도 75%+ → 첫 피스 SMALL 보장 + FIT 체크 90% (CRITICAL)
100% 풀  → Rescue 시스템 → SINGLE 강제 (빈 셀 있으면)
tight 트레이 2회 연속 → Relief 트레이 (TINY+SMALL+SMALL)
```

**Continue (광고 보상):** 하단 3행 제거 + 트레이 리필

### 4.3 게임오버 판정 (game_over_system.gd)

```gdscript
not board.can_place_any_piece(all_pieces)
```
- 트레이 피스 + 홀드 피스(미사용 시) 모두 배치 불가능해야 게임오버
- **홀드 시스템이 "마지막 한 수" 역할** — 좋은 설계

### 4.4 "아쉬운" vs "짜증나는" 분석

**🟢 "아쉬운" 요소:**
- Mercy 시스템이 4단계로 점진적 → "기회를 줬는데 못 살렸다" 느낌
- Relief 트레이가 연속 실패 후 제공 → "한 번 더 기회" 느낌
- 홀드 시스템 → "내가 더 잘 판단했으면..." 느낌
- Pre-fill 30% → 게임 시작부터 빈 보드가 아니라 "이미 진행 중" 느낌

**🔴 "짜증나는" 잠재 요소:**
- **Near-miss 힌트 비활성** → 7/8 채워진 행을 인지 못하고 놓칠 수 있음
- **FIT 체크가 확률적** (70~90%) → 가끔 배치 불가 피스가 나올 수 있음 (10~30%)
- **게임오버 SFX가 하강 5음(550ms)으로 단조로움** — 아쉬움보다 "끝" 느낌
- **게임오버 햅틱이 100ms 단일 펄스** — 임팩트 부족

### 4.5 개선 제안

| 항목 | 현재 | 제안 | 이유 |
|------|------|------|------|
| Near-miss 힌트 | 비활성 | **재활성화** | "거의 완성" 인지 → 아쉬움 극대화 |
| FIT 체크 CRITICAL | 90% | **100%** | 밀도 75%+ 에서 배치 불가 피스 = 분노 유발 |
| 게임오버 연출 | 즉시 전환 | **0.5초 슬로모션 + 보드 회색화** | "아..." 순간 연출 |
| 게임오버 햅틱 | 100ms 단일 | **200ms + 150ms(+200ms)** 이중 펄스 | 심장 뛰는 느낌 |
| 게임오버 SFX | 하강 5음 | 하강 후 **작은 "띵" 음** 추가 | "또 해볼까" 유인 |
| 위기 경고 | 보더 펄스만 | **BGM 변화 + 셀 미세 진동** | 긴장감 다채널 전달 |

## 5. 속도감 분석

### 5.1 피스 배치 → 클리어 타임라인

```
T=0ms      피스 배치 (block_place SFX, -14dB, 80ms)
T=0ms      스코어 팝업 즉시 생성
T=0ms      배치 펄스 시작 (150ms)
T=0ms      보드 바운스 (클리어 없을 때만, 120ms)

--- 클리어 발생 시 ---
T=100ms    라인 클리어 이펙트 시작
             - Hit Stop 20~40ms
             - 셀 플래시 340ms (셀당 20ms 스태거)
             - 파티클 버스트 1000ms
             - 쇼크웨이브 400ms
             - 스크린 셰이크 80~180ms
             - 라인 클리어 SFX 250ms
             - 햅틱 55~80ms+

T=350ms    콤보 팝업 (x2+ 시, 820ms 총 시간)
T=350ms    콤보 SFX 80~300ms
T=350ms    콤보 햅틱 100~250ms

T=500ms    체인 이펙트 (있을 시)
             - Hit Stop 40ms
             - 캐스케이드 셀 팝 (그룹당 100ms 간격)
             - 체인 SFX 250ms
             - 스크린 셰이크 120ms

T=700ms    블래스트 이펙트 (있을 시)
             - Hit Stop 60ms
             - 화면 플래시 200ms
             - 셀 폭발 150ms
             - 쇼크웨이브 400ms
             - 스크린 셰이크 250ms
             - 블래스트 SFX 350ms

T=800ms+   퍼펙트 클리어 (있을 시)
T=900ms+   레벨 업 (있을 시)
```

### 5.2 총 이펙트 시간

| 시나리오 | 총 시간 | 플레이어 대기 시간 | 평가 |
|---------|---------|-----------------|------|
| 단순 배치 (클리어 없음) | ~150ms | **0ms** (즉시 다음 조작) | ✅ 빠름 |
| 1라인 클리어 | ~450ms | **~100ms** (이펙트는 비차단) | ✅ 양호 |
| 2라인 + 콤보 | ~700ms | **~100ms** | ✅ |
| 3라인 + 체인 + 블래스트 | ~1200ms | **~100ms** | ✅ |
| 퍼펙트 클리어 풀 연출 | ~2000ms | **~100ms** | ✅ |

**핵심 발견:** 이펙트가 **비차단(non-blocking)**임. 보드 상태 업데이트는 즉시 완료되고, 시각/청각 이펙트는 타이머 기반으로 나중에 재생됨. 플레이어는 이론상 이펙트 진행 중에도 다음 피스를 잡을 수 있음.

**그러나:** Hit Stop(Engine.time_scale=0)이 20~60ms 적용되며, 이 동안 드래그 입력이 처리되지 않음. 연속 Hit Stop이 중첩되면 체감 지연 발생 가능.

### 5.3 "빠른 연속 배치" 쾌감

**현재 구조:**
1. 피스 배치 → board_renderer.update_from_state() 즉시
2. 트레이 비면 → _refill_tray() 즉시 (populate_tray에 appear 애니메이션)
3. 새 피스 드래그 가능

**문제:**
- 트레이 리필 시 `populate_tray(new_tray, true)` — appear 애니메이션이 있지만, 플레이어가 즉시 드래그 시작 가능한지 코드에서 블록 없음
- **Hit Stop이 누적되지 않음** (더 긴 것만 적용) — ✅ 올바른 설계
- **이펙트 시퀀스가 `ignore_time_scale=true` 타이머 사용** — Hit Stop 중에도 이펙트 시퀀스 진행 ✅

### 5.4 리듬 방해 요소

**🟢 양호:**
- 모든 이펙트가 비차단 — 게임 리듬을 깨지 않음
- Hit Stop이 짧고(20~60ms) 누적 불가 — 체감 지연 최소
- Score Cascade의 wall-clock 타이밍 — hit stop과 독립

**🔴 잠재 문제:**
- 콤보 팝업 홀드 시간 **450ms**가 다소 길 수 있음 (시각적 방해)
- 블래스트 팝업 홀드 시간 **600ms** — 화면 중앙을 오래 차지
- 파티클 수명 **1000ms** — 빠른 연속 배치 시 이전 파티클이 아직 남아 시각 혼잡

### 5.5 속도감 개선 제안

| 항목 | 현재 | 제안 | 이유 |
|------|------|------|------|
| 콤보 팝업 홀드 | 450ms | **300ms** | 시각 방해 감소 |
| 블래스트 팝업 홀드 | 600ms | **400ms** | 빠른 게임 리듬 |
| 파티클 수명 | 1000ms | **700ms** | 시각 혼잡 방지 |
| 라인 클리어 셀 스태거 | 20ms/셀 | **15ms/셀** | 8셀 = 160ms→120ms, 더 빠른 클리어 |
| Hit Stop (1라인) | 20ms | **15ms** | 미세 차이지만 빠른 리듬 |
| 이펙트 시작 딜레이 | 100ms | **60ms** | 배치→클리어 반응 빠르게 |

## 7. Block Blast 대비 차별화 전략

ChromaBlocks만의 독자적 강점:
1. **Chroma Chain/Blast** — Block Blast에 없는 연쇄 시스템으로 "예측 불가 잭팟" 제공
2. **색상 전략** — 단순 라인 클리어를 넘는 2차원 목표
3. **Hit Stop** — 격투게임에서 가져온 "임팩트" 기법
4. **합성 SFX** — 에셋 의존 없이 독특한 레트로 사운드

활용되지 않는 무기:
1. **Color Match** (비활성) — 켜면 Block Blast와 완전히 다른 게임
2. **Score Cascade** (미사용) — 점수 분해 쾌감은 퍼즐 게임의 핵심 중독 요소
3. **Anticipation Phase** (미사용) — "곧 터진다!" 기대감
4. **Near-miss 힌트** (비활성) — "거의 완성!" 계속 플레이 유인

**결론: 코드에 이미 존재하는 미사용 기능만 활성화해도 중독성이 크게 향상될 수 있음.**
