# ChromaBlocks 중독성 분석 — Game Designer 관점

> 게임 메커니즘 레벨에서 "계속 하고 싶지 않은" 원인 분석
> 작성일: 2026-03-25

---

## 요약: 핵심 진단

ChromaBlocks는 Block Blast의 검증된 Core Loop에 Chroma Chain과 Chroma Blast라는 두 개의 부가 시스템을 얹었다. **문제는 이 부가 시스템이 중독성을 높이는 게 아니라 오히려 Core Loop의 "순수한 만족감"을 희석시키고 있다는 점이다.**

Block Blast가 중독적인 이유는 **의사결정의 명쾌함** 때문이다. "여기 놓으면 줄이 클리어된다/안 된다" — 끝. ChromaBlocks는 여기에 "혹시 같은 색 5개 모이면 체인이 터지고, 한 줄에 같은 색 6개면 블라스트가 터지고..."라는 **예측 불가능한 보너스 레이어**를 추가했는데, 이것이 플레이어의 **에이전시(agency) 감각**을 약화시킨다.

**한 문장 진단: "뭘 해야 잘하는 건지 모르겠는 게임은 중독되지 않는다."**

## 1. Core Loop 분석

### 현재 Core Loop 구조

```
피스 선택 → 배치 위치 결정 → 클리어 확인 → [크로마 체인] → [크로마 블라스트] → 점수 → 새 피스
   │              │              │              │                │
   ▼              ▼              ▼              ▼                ▼
 선택지 3개    그리드 전체    자동 판정     자동 판정          자동 판정
 (의사결정)    (의사결정)    (수동적 관찰)  (수동적 관찰)     (수동적 관찰)
```

### 각 단계별 의사결정 분석

| 단계 | 플레이어 의사결정 | 깊이 | 재미 |
|------|------------------|------|------|
| 피스 선택 | 3개 중 어떤 것을 먼저 놓을까 | ★★★ | ★★★ — 순서 전략이 존재 |
| 배치 위치 | 어디에 놓아야 줄이 완성되나 | ★★★★ | ★★★★ — Core 재미, 공간 퍼즐 |
| 줄 클리어 | 없음 (자동) | - | ★★★ — 보상감 |
| 크로마 체인 | 없음 (자동, 예측 거의 불가) | - | ★☆☆ — "어 뭔가 터졌네" |
| 크로마 블라스트 | 없음 (자동, 예측 가능하나 우연적) | - | ★★☆ — 터지면 시원하지만 내가 한 게 아닌 느낌 |

### 코드에서 확인한 Core Loop의 문제

**`chroma_blocks_game.gd`의 `_place_piece()` 함수가 400줄 넘게 거대하다.** 이는 단순히 코드 품질 문제가 아니라, **하나의 배치 액션에 너무 많은 시스템이 연쇄적으로 걸려있다**는 게임 디자인 문제를 반영한다:

1. `place_piece` → 보드 업데이트
2. `get_completed_rows/cols` → 줄 완성 체크
3. `ChromaBlastSystem.check_blast` → 같은 색 6+ 체크
4. `ClearSystem.check_and_clear` → 줄 클리어
5. `ChromaBlastSystem.execute_blast` → 블라스트 실행, 추가 줄 클리어
6. `ChromaChainSystem.process_chains` → 5+ 연결 체크, 추가 줄 클리어
7. `ColorMatchSystem.check_color_match` → (비활성이지만 구조 존재)
8. `SpecialTileSystem.try_drop_specials` → 특수 타일 드롭
9. `ScoringSystem.calculate` → 점수 계산

**플레이어가 블록 하나를 놓으면 최대 9단계의 시스템이 연쇄적으로 동작한다.** 이 중 플레이어가 의도하고 예측할 수 있는 것은 1-4번까지다. 5-9번은 "보너스"이지만, 실제로는 **"내가 의도한 결과가 아닌 무언가가 일어났다"**는 경험이 된다.

## 2. 경쟁작 Core Loop 비교

### Block Blast: 왜 이것만으로 중독인가?

```
피스 선택 → 배치 → 줄 클리어 → 점수 → 새 피스
   │          │         │
   ▼          ▼         ▼
  3개 중 1   어디에?   "내가 했다!"
```

Block Blast의 중독 공식:

1. **완벽한 인과관계**: "여기 놓으면 줄이 클리어된다" → 놓는다 → 줄이 클리어된다. 끝. 100% 예측 가능하고 100% 플레이어의 성취다.
2. **짧은 보상 루프**: 의사결정(2-5초) → 결과(즉시) → 보상감(0.3초) → 다음 의사결정. 이 사이클이 **8초 이내**로 반복된다.
3. **"거의 됐는데" 순간의 빈도**: 8x8 그리드에서 7/8칸이 채워진 줄을 보면 "한 칸만 더!" 이 순간이 자연스럽게 자주 발생한다.
4. **단순한 스킬 표현**: 좋은 배치 = 더 많은 줄 클리어 = 높은 점수. 명확하다.
5. **게임 오버의 점진성**: 보드가 서서히 차면서 "이거 큰일인데..." → 마지막 순간까지 긴장감.

### 1010!: 같은 구조인데 왜 성공했나?

1010!은 10x10 그리드에 같은 구조를 가졌지만 **피스 다양성이 핵심**이었다. 다양한 크기와 모양의 피스가 "이거 어디에 맞추지?"라는 공간 퍼즐의 재미를 극대화했다. 그리고 **부가 시스템이 없다**. 줄 클리어가 전부이고, 그것만으로 충분하다.

### ChromaBlocks의 크로마 시스템: 방해 요인인가?

#### Chroma Chain (5+ 같은 색 연결)

코드 분석 (`chroma_chain_system.gd`):
- `CHROMA_CHAIN_THRESHOLD = 5` — 같은 색 5개 연결
- `CHROMA_CHAIN_MAX_CASCADE = 2` — 최대 2회 연쇄
- 줄 클리어 **후**에 발동 → 클리어로 인해 떨어진 블록이 5개 연결되면 추가 폭발

**문제점:**
- 플레이어가 "줄 클리어 이후 남은 블록의 색 배치"를 미리 계산하기 극히 어렵다
- 즉, 체인은 **대부분 우연히** 터진다
- 우연한 보상은 도파민을 유발하지만, **"내가 잘해서 터뜨렸다"는 감각이 없다**
- 슬롯머신과 비슷한 패턴 → 퍼즐 게임 플레이어가 원하는 감정이 아님

#### Chroma Blast (한 줄에 같은 색 6+)

코드 분석 (`chroma_blast_system.gd`):
- `CHROMA_BLAST_THRESHOLD = 6` — 한 줄에 같은 색 6개 이상
- 줄 클리어 **전**에 체크 → 해당 색 전체 보드에서 제거

**문제점:**
- 8칸 줄에서 같은 색 6개는 **플레이어가 의도적으로 만들기 거의 불가능**
- 6가지 색상 × 8칸 = 확률적으로 가능하지만, 다른 피스의 색은 랜덤이므로 전략적 배치가 아닌 **운의 영역**
- 블라스트가 터지면 시각적으로는 화려하지만, "왜 터졌지?"라는 의문이 먼저 든다
- 보드 위의 같은 색 전체 제거 → 이후 보드 상태 예측 불가 → **전략적 플레이 방해**

#### 핵심 비교표

| 요소 | Block Blast | ChromaBlocks | 영향 |
|------|-------------|-------------|------|
| 배치 → 결과 인과관계 | 100% 예측 가능 | ~60% 예측 가능 | 에이전시 약화 |
| 보상 루프 길이 | 3-8초 | 5-15초 (연출 포함) | 몰입 저하 |
| 스킬 표현 | 명확 (줄 클리어 = 실력) | 불명확 (운 + 실력 혼합) | 재플레이 동기 약화 |
| "아, 알겠다!" 순간 | 자주 (줄 거의 완성 시) | 드묾 (체인/블라스트 예측 어려움) | 학습 곡선 문제 |
| 이펙트 시퀀스 | 200-300ms | 500-1500ms | 다음 수 대기 시간 |

## 3. 난이도 곡선 분석

### game_constants.gd 파라미터 분석

```gdscript
# 레벨업 공식
static func lines_for_next_level(level: int) -> int:
    return mini(level * 5, 50)

# 즉: Lv1→5줄, Lv2→10줄, Lv3→15줄, ... Lv10+→50줄
```

**문제: 초반 레벨업이 너무 빠르다.**
- Lv1 → 5줄만 클리어하면 레벨업 (3-4회 배치)
- 레벨업 자체가 주는 게임플레이 변화가 미미함 (피스 종류 제한 완화 뿐)
- Block Blast는 레벨 개념 없이 **점수 자체가 진행도** → 더 명쾌

### DDA(Dynamic Difficulty Adjustment) 분석

`piece_generator.gd`의 DDA 시스템:

```
fill < 20% → RUSH 모드 (큰 피스, 50% 확률)
fill 40%+  → MERCY MILD (작은 피스 가중치 1.8x)
fill 60%+  → MERCY STRONG (작은 피스 가중치 3.0x)
fill 75%+  → MERCY CRITICAL (최소 1개 SMALL 피스 보장)
fill 100% → 게임 오버
```

**문제점:**

1. **머시가 너무 강하다.** fill 60%에서 이미 작은 피스 가중치 3x → 위기감 부족. Block Blast는 "좋은 피스가 올 때도 있고 나쁜 피스가 올 때도 있다"의 **긴장감**이 핵심인데, ChromaBlocks는 위기 시 구조대가 너무 빨리 온다.

2. **Anti-frustration이 과도하다.**
   - `MAX_TIGHT_STREAK = 2` → 답답한 트레이 2번 연속이면 무조건 구제
   - `_rescue_tray()` → 배치 불가능한 트레이 자동 수정
   - `_generate_relief_tray()` → TINY+SMALL+SMALL 구제 트레이

   이 모든 안전망이 **"이번엔 진짜 망했다..."라는 스릴을 제거**한다.

3. **RUSH 모드의 역설.** 보드가 비면 큰 피스를 줘서 빨리 채우는 건 좋은 아이디어지만, 실행에 문제가 있다. fill < 20%는 게임 시작 직후이거나 대규모 클리어 직후인데, 이 시점에서 큰 피스를 받으면 "쉽고 지루한" 구간이 길어진다.

### "달성 가능하지만 아슬아슬한" 순간이 있는가?

**거의 없다.** 이유:

1. 머시 시스템이 너무 관대해서 진짜 위기가 드묾
2. 크로마 블라스트가 예측 불가능하게 보드를 정리해버림 → 위기 해소가 "내 실력"이 아닌 "시스템 개입"
3. 30% 프리필로 시작하는 보드(`_prefill_board`) → 초반부터 이미 어지러운 상태이지만, 플레이어가 이해할 수 있는 구조가 아님 (랜덤 색상 배치이므로)

**Block Blast의 "아슬아슬함":**
- 보드가 서서히 차면서 "이 피스만 맞추면..." → 줄 클리어 → 안도감
- 이 텐션 커브가 자연스럽게 반복됨
- 머시 없음 — 운 나쁘면 진짜 게임오버 → 다음 판에서 더 잘해야지

## 4. 전략적 깊이 분석

### "나만의 전략"을 개발할 수 있는가?

Block Blast에서 고수들이 쓰는 전략:
1. **코너 전략**: 한쪽 코너를 비워두고 나머지를 채움
2. **줄 우선 전략**: 거의 찬 줄을 빨리 완성시키는 순서
3. **보드 관리**: 빈 공간을 한쪽에 모으는 전략

ChromaBlocks에서의 전략 가능성:
1. ✅ 줄 완성 우선 배치 — 가능하지만 Block Blast와 동일
2. ❓ 같은 색 모으기 — 체인 유도? 가능하지만 효과가 너무 불확실
3. ❓ 블라스트 유도 — 한 줄에 같은 색 6개? 사실상 불가능
4. ❌ 홀드 시스템 활용 — 있지만, 1트레이 1회 제한으로 영향 미미

**결론: 크로마 시스템이 추가한 전략적 깊이보다, 그것이 가져온 예측 불가능성이 더 크다.**

### 고수와 초보의 점수 차이

AutoPlayer(`auto_player.gd`) 코드에서 봇의 평가 함수가 보여주는 것:
- 줄 완성 (W_LINE_CLEAR = 100)
- 인접성 (W_ADJACENCY = 10)
- 구멍 회피 (W_HOLE_PENALTY = -20)
- 하단 우선 (W_BOTTOM_PREFER = 5)

봇조차 **크로마 체인이나 블라스트를 전혀 고려하지 않는다.** 이는 이 시스템들이 전략적 의사결정의 대상이 아님을 방증한다.

### 스킬 표현의 여지

| 스킬 | Block Blast | ChromaBlocks |
|------|-------------|-------------|
| 공간 효율 배치 | ★★★★★ | ★★★★ (동일) |
| 줄 완성 계획 | ★★★★★ | ★★★★ (동일) |
| 위기 관리 | ★★★★ | ★★☆ (머시가 대신 해줌) |
| 콤보 설계 | ★★★ | ★★☆ (동일하나 이펙트 시퀀스가 방해) |
| 고유 시스템 활용 | N/A | ★☆☆ (체인/블라스트는 운) |

## 5. 세션 구조 분석

### 한 판 길이

- **Block Blast 평균 세션**: 3-7분
- **ChromaBlocks 예상 세션**: 5-12분 (이펙트 시퀀스 + 프리필 시작 + 강한 머시로 생존 연장)

**문제: 한 판이 길면 "한 판 더"가 줄어든다.**

캐주얼 퍼즐의 황금 법칙: **"아쉬울 때 끝나야 다시 한다."**
- 3분 만에 죽으면 → "아 한 판 더!" (감정적 즉시 재도전)
- 10분 만에 죽으면 → "음... 다음에" (투자 대비 만족 계산 시작)

### 게임오버의 특성

**현재 구조 (`game_over_system.gd`):**
- 트레이의 모든 피스(+홀드 피스)가 배치 불가 → 즉시 게임오버

**문제:**
- 머시 시스템이 게임오버 직전에 구제 트레이를 주므로, 실제 게임오버는 **갑자기** 온다
- 즉, "서서히 위기감이 고조되다가 마지막까지 발버둥 치는" 경험이 아니라
- "계속 살아남다가 갑자기 끝" → **감정적 여운이 없음**
- Block Blast는 머시가 없어서 보드가 차면 **진짜 위기** → 마지막 몇 수가 극도로 긴장감 → 게임오버 후 "아... 그때 그렇게 놨으면 됐는데!" → 즉시 재도전

### "한 판 더" 유도 메타 구조

현재 ChromaBlocks에 있는 메타 구조:
1. ✅ 하이스코어 갱신 → 있음 (game_over_screen에 NEW BEST 표시)
2. ✅ 그레이드 시스템 → S/A/B/C 등급 (10000/5000/2000)
3. ✅ 일일 챌린지 → 있음 (daily_challenge_system)
4. ✅ 미션 시스템 → 있음 (mission_system)
5. ❌ 프로그레션 시스템 → **없음** (XP를 모아서 뭔가 언락하는 구조 부재)
6. ❌ 소셜 비교 → **없음** (리더보드 부재)
7. ❌ "한 판 더" 당기는 중간 목표 → **없음**

**핵심 부재: 미션 XP를 모아도 갈 곳이 없다.**
- 미션 완료 → XP 획득 → ??? → 보상 없음
- 프로그레션 없이 XP는 의미 없는 숫자
- "다음 미션 보너스 언락까지 50XP!" 같은 당근이 없음

## 6. 개선안: 코드 레벨 제안

### A. Core Loop 단순화 — "Less is More"

**제안: 크로마 체인 & 블라스트를 기본 모드에서 제거하고, 별도 "크로마 모드"로 분리**

```gdscript
# game_constants.gd 수정
const CHROMA_CHAIN_ENABLED: bool = false  # 기본 모드에서 비활성
const CHROMA_BLAST_ENABLED: bool = false  # 기본 모드에서 비활성
```

이렇게 하면:
- Core Loop가 `배치 → 줄 클리어 → 점수 → 새 피스`로 단순화
- Block Blast와 동일한 검증된 루프
- `_place_piece()` 함수에서 5-8번 단계가 스킵되어 이펙트 시퀀스도 짧아짐
- "크로마 모드"는 고급 유저를 위한 별도 게임 모드로 제공

### B. 머시 시스템 약화 — "긴장감 복원"

```gdscript
# piece_generator.gd 수정 제안
const DDA_MERCY_MILD := 0.55       # 40% → 55% (더 늦게 머시 시작)
const DDA_MERCY_STRONG := 0.70     # 60% → 70%
const DDA_MERCY_CRITICAL := 0.85   # 75% → 85%
const DDA_FIT_CHECK_CHANCE := 0.40 # 70% → 40% (운에 더 맡김)
const MAX_TIGHT_STREAK: int = 4    # 2 → 4 (더 오래 버텨봐)
```

**효과:**
- 보드 50-70% 구간에서 진짜 위기감 발생
- "이번 트레이로 살아남을 수 있을까?" 긴장감
- 게임오버가 덜 갑작스럽고 더 점진적

### C. 이펙트 시퀀스 압축 — "빠른 보상 루프"

현재 `_play_effects_sequence()`의 타임라인:
```
0ms    — 점수 팝업
100ms  — 줄 클리어 + 쉐이크
350ms  — 콤보 팝업
500ms  — 체인 이펙트
700ms  — 블라스트 이펙트
~1000ms — 완료
```

제안 (크로마 시스템 유지 시):
```
0ms    — 점수 팝업 + 줄 클리어 (동시!)
150ms  — 콤보 (겹침)
250ms  — 체인+블라스트 (합쳐서 한 이벤트)
~400ms — 완료
```

**핵심: 배치 후 다음 의사결정까지 최대 400ms. Block Blast 수준.**

### D. 프로그레션 시스템 추가 — "한 판 더의 이유"


이렇게 하면:
- 미션 XP에 의미가 생김
- "레벨 5까지 올리면 홀드 업그레이드!" → 플레이 동기
- 크로마 모드를 메타 보상으로 → 고급 유저 전용 콘텐츠화

### E. 보드 프리필 제거 또는 최소화

```gdscript
# chroma_blocks_game.gd의 _prefill_board() 수정
func _prefill_board(board: BoardState) -> void:
    pass  # 빈 보드에서 시작
```

**이유:**
- Block Blast/1010!은 빈 보드에서 시작
- 빈 보드 → 첫 몇 수가 "무료"로 안전함 → 자연스러운 온보딩
- 30% 프리필된 랜덤 보드는 "이미 어지러운 상태에서 시작" → 초보에게 부담
- 색상 클러스터 프리필은 크로마 체인을 위한 것이지만, 크로마 시스템 자체가 문제

### G. 색상 전략 강화 (크로마 시스템 유지 시)

만약 크로마 시스템을 유지하기로 한다면, **플레이어가 색을 전략적으로 활용할 수 있어야** 한다:

```gdscript

# 아이디어: 트레이의 피스 색을 탭하면 변경 가능 (1트레이 1회)
# 또는: 피스 색상을 2가지 옵션에서 선택
```

이렇게 하면:
- 체인/블라스트가 운이 아닌 **전략**이 됨
- "이 피스를 빨간색으로 바꾸면 체인이 터진다!" → 의미 있는 의사결정
- 스킬 표현 여지 대폭 증가

## 결론

ChromaBlocks의 근본적 문제는 **"차별화를 위해 추가한 것이 오히려 핵심을 약화시킨 것"**이다.

Block Blast가 증명한 건 **"완벽한 단순함"**의 힘이다. 피스를 놓고 줄을 클리어하는 것 — 이 루프가 수십 년간 중독적인 이유는, 플레이어가 **모든 결과를 자신의 의사결정으로 귀인(attribution)**할 수 있기 때문이다.

크로마 체인과 블라스트는 "와, 뭔가 터졌다!"를 만들지만, "내가 해냈다!"를 만들지 못한다. 후자가 없으면 중독은 없다.

**가장 중요한 한 마디: 크로마 시스템을 끄고, Block Blast 수준의 순수한 Core Loop로 돌아가라. 차별화는 비주얼과 메타(프로그레션, 미션, 일일 챌린지)로 하라. 메커니즘이 아니라.**
